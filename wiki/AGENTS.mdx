<h1>Wiki Agent Guide</h1>
<p>This directory is the maintained MDX-first knowledge and planning layer for hyperwiki.</p>
<h2>Rules</h2>
<ul>
  <li>Read <a href="/wiki/index.mdx">index.mdx</a> before structural wiki changes.</li>
  <li>Use <a href="/wiki/sources.mdx">sources.mdx</a> as the source index.</li>
  <li>Use <a href="/wiki/scaffold-contract.mdx">scaffold-contract.mdx</a> before normalizing external project-wiki skill conventions into hyperwiki.</li>
  <li>Use <a href="/wiki/plans/index.mdx">plans/index.mdx</a> before meaningful implementation work.</li>
  <li>Preserve hyperwiki scaffold conventions: lowercase <code>wiki/sources.mdx</code>, app-visible <code>wiki/AGENTS.mdx</code>, and app-served <code>/assets/wiki.css</code>.</li>
  <li>For React, Tailwind, shadcn/ui, or MDX plan code, follow the direction captured in the completed <a href="/wiki/plans/zzz_completed/features/react-shadcn-mdx-rewrite.mdx">React, shadcn, and MDX Rewrite</a> and use the repo-local <code>.agents/skills/shadcn</code> and <code>.agents/skills/tailwind-design-system</code> skills before making changes.</li>
  <li>For visible wiki plan examples, prefer <code>CodeBlock</code> over raw fenced code blocks when a title, language label, copy affordance, or tabbed alternatives would help. Compose <code>Tabs</code>/<code>Tab</code> with one <code>CodeBlock</code> per tab for alternate files, languages, clients, or package-manager examples.</li>
  <li>Keep runtime state out of the wiki unless it is intentionally exported.</li>
</ul>
