Design Brief

Status

  • Last reviewed: 2026-05-11
  • Evidence basis: product definition and dogfood plan
  • Confidence: medium

Interface Principles

  • Dense, quiet, work-focused local development surface.
  • Wiki navigation and terminal panels must remain scannable together.
  • Agent automation must be visible as a named terminal panel and driven by explicit project configuration.
  • Repo files and Git are canonical, so the UI should reveal file paths and runtime boundaries.
  • The Dashboard should stay compact and work-focused, surfacing ideas and projects as operational lists rather than a marketing-style landing page.
