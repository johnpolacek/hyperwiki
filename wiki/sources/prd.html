Product Intent

hyperwiki is Localhost Tooling for docs-driven agentic development. It turns repo-local project docs, plans, logs, verification status, terminals, and agent sessions into a local development environment whose trust boundary is the developer's own machine, files, credentials, and environment variables.

Primary Flow

  1. Install or invoke the CLI.
  2. Initialize an HTML wiki from repository context.
  3. Run the local dev workspace.
  4. Use wiki pages and terminal panels to drive agentic development transparently.

Product Boundary

hyperwiki should not reposition itself as a hosted dashboard with a local mode. Local execution, repo-visible truth, explicit runtime state, and inspectable agent handoff are the product thesis.
