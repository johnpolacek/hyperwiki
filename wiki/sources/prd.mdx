<h2>Product Intent</h2>
<p>hyperwiki is Localhost Tooling for docs-driven agentic development. It turns repo-local project docs, plans, logs, verification status, terminals, and agent sessions into a local development environment whose trust boundary is the developer's own machine, files, credentials, and environment variables.</p>
<h2>Primary Flow</h2>
<ol>
  <li>Install or invoke the CLI.</li>
  <li>Initialize an HTML wiki from repository context.</li>
  <li>Run the local dev workspace.</li>
  <li>Use wiki pages and terminal panels to drive agentic development transparently.</li>
</ol>
<h2>Product Boundary</h2>
<p>hyperwiki should not reposition itself as a hosted dashboard with a local mode. Local execution, repo-visible truth, explicit runtime state, and inspectable agent handoff are the product thesis.</p>
