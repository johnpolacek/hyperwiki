<h1>Architecture</h1>
<p>hyperwiki is Localhost Tooling packaged as a Tauri desktop app with a Rust-owned local runtime, repo-local canonical wiki files, and Rust CLI commands. The legacy Node localhost server has been removed; the desktop app is the canonical runtime.</p>
<p>Its trust boundary is the developer's own machine, repo files, Git state, terminal sessions, credentials, and environment variables. Hosted services are optional integrations, not the source of truth.</p>
<h2>Boundaries</h2>
<ul>
  <li>The Rust CLI performs repeatable filesystem operations and starts the desktop runtime.</li>
  <li>The wiki is canonical repo-visible HTML.</li>
  <li>Projects are the primary workspace unit; imported briefs create registered hyperwiki projects directly.</li>
  <li>The Tauri runtime renders the wiki and terminal workspace but is not a hidden source of truth.</li>
  <li>A user-level registry at <code>~/.hyperwiki/projects.json</code> tracks known local hyperwiki projects for switching.</li>
  <li>Runtime state is ignored under <code>.hyperwiki/state/</code> and <code>.hyperwiki/sessions/</code>.</li>
  <li>Multi-project runtime APIs resolve project roots through the registry, but each project's wiki, Git status, and terminal sessions remain isolated to that project root.</li>
  <li>The workspace exposes local-only mode, canonical wiki/Git truth, ignored runtime state, session metadata, command history boundaries, and export/prune actions through the guardrails UI and <code>/api/guardrails</code>.</li>
  <li>The Projects page is a workspace route rather than a wiki file; it reads registered projects and hands brief-driven project creation work to an agent terminal.</li>
  <li>The Execute control is the primary start-work boundary: it creates terminals lazily and posts scoped main/worktree instructions into the visible agent session.</li>
  <li>Current structured surfaces are served through the Tauri command transport. Verification loops, project contracts, review workflows, MCP resources, and terminal sessions are Rust-owned runtime surfaces.</li>
</ul>
<h2>Localhost Tooling Commitments</h2>
<ul>
  <li>Bind local runtime surfaces to localhost addresses by default.</li>
  <li>Keep canonical project knowledge in repo-visible files and Git.</li>
  <li>Keep runtime state local and ignored unless a user intentionally exports or records it.</li>
  <li>Expose permission, session, command-history, and export boundaries in the workspace instead of hiding them in app state.</li>
  <li>Avoid default hosted sync, telemetry, remote execution, or cloud-backed knowledge stores.</li>
</ul>
