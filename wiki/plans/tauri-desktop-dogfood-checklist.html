Manual Desktop Dogfood Checklist

Summary

  • Status: active checklist
  • Plan: Complete Tauri Rewrite
  • Target: macOS first, then cross-platform parity when packaging reaches Stage 05.

Launch And App Shell

  • Start with pnpm run tauri:dev and confirm the dedicated desktop window opens without browser chrome.
  • Confirm app-shell navigation opens Projects, Settings, Planning Dashboard, and wiki pages without losing the active workspace route.
  • Confirm external preview links open outside the app window and local-only guardrails remain visible.

Project And Wiki Data

  • Open an existing registered project and verify projects, wiki, settings, and Git state match the Node reference app.
  • Create a project from a brief and verify generated wiki files, source briefs, roadmap, and plan dashboard are repo-visible.
  • Remove a project checkout and verify stale registry entries are handled without deleting files unless explicitly requested.

Terminal, Sessions, And Previews

  • Start agent and CLI terminals, then verify terminals, sessions, and plan-scoped reattach behavior.
  • Run a Portless dev preview, verify previews use exact checkout hostnames, and confirm Run Dev captures runtime URLs.
  • Create a worktree and verify Git branch state, preview URL, terminal scope, and reload persistence.

Verification, Reviews, And MCP

  • Open verification and project-contract data and compare with the Node reference outputs.
  • Prepare reviews for architecture, security, diff, and test gaps; verify review prompts route only to active agent sessions.
  • Run the mcp transport smoke and verify mcp resources stay local and permissioned.
