<section>
  <h2>Intent</h2>
  <p>Theme editing should feel direct. Choosing a preset or changing theme controls should update the running app immediately and save without requiring a separate Save Theme action.</p>
</section>
<section>
  <h2>Unit 01 - Theme editor autosave</h2>
  <ul>
    <li>Apply theme drafts to the app shell as soon as they change.</li>
    <li>Debounce writes to the existing global settings endpoint so rapid color and select changes do not spam persistence.</li>
    <li>Keep a baseline from the moment the theme editor opens so Revert can restore that theme and save the restored value.</li>
    <li>Remove the Save Theme action, rename Cancel to Revert, and provide Done as the non-saving exit that flushes pending autosave work before returning to Settings.</li>
  </ul>
</section>
