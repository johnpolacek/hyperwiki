Intent

Tie the visible terminal workspace to the active plan root so each plan opens with its own right side, while terminal processes for other plans, projects, and worktrees keep running locally until explicitly closed.

Implementation

  • Resolve terminal scope from the active route: feature plans use their plan page, MVP units use their parent stage, and overview or non-plan routes use their own runtime scopes.
  • Detach terminal UI on scope changes without deleting or killing server sessions.
  • Keep PTY and pipe fallback processes alive after WebSocket disconnects, with bounded replay for reattached browser panels.
  • Route agent prompts and review workflow submissions to the current scope's active agent session.
  • Keep session metadata under ignored .hyperwiki/sessions/ and preserve the runtime-only export boundary.

Verification

  • Different plans show independent terminal sets.
  • Returning to a plan reattaches the original terminal process.
  • Explicit close still terminates a terminal and removes it from future restores.
  • Project and worktree switches preserve isolation by project root.
  • Verified passing checks: pnpm run check, pnpm run smoke:pty, pnpm run smoke:sessions, and pnpm run smoke:launch.
  • Known unrelated blocker: pnpm run smoke:browser currently fails on a completed-stage dot color expectation before terminal-scope assertions complete.
