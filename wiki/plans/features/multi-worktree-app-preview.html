Multi-Worktree App Preview

Summary

  • Status: complete
  • Shape: compact feature plan
  • Current unit: complete
  • Validation: syntax check, browser smoke, preview status smoke

Intent

Make app preview launching exact-checkout aware across main and registered worktrees. Hyperwiki should never open a main preview when the active workspace is a worktree, and it should not present a dead URL as a working app.

Unit 01 - Exact Checkout Preview Launcher

  • Status: complete
  • Add preview status APIs that resolve exact URLs for main and registered worktrees.
  • Use exact Portless hostname matching to distinguish running, stopped, unknown, and unconfigured previews.
  • Expose all registered worktrees in the project switcher and Manage Projects UI.
  • Make Open App start the active checkout's dev terminal when needed, poll readiness, and open only the exact checkout URL.

Verification
