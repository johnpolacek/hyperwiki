<Scope>
  <p>Normalize standalone product prose from mixed-case spellings to <code>hyperwiki</code> in live UI copy, Rust/Tauri runtime strings, generated project scaffolds, agent prompts, bundled skill guidance, README, and all wiki history.</p>
  <p>Preserve technical identifiers and compatibility strings where casing is part of the interface: TypeScript and Rust type names, environment variables, managed comment markers, package names, command names, crate names, and paths such as <code>.hyperwiki</code>.</p>
</Scope>

<Decision title="Canonical spelling">
  <p>The product brand is lowercase <code>hyperwiki</code> everywhere it appears as human-readable product language, including sentence starts and headings.</p>
</Decision>
