Git Onboarding

Summary

  • Status: complete
  • Shape: compact product safety correction
  • Current unit: complete
  • Validation: pnpm run check, init smoke, project links smoke, app preview smoke, project removal smoke, launch smoke, browser smoke

Intent

Hyperwiki should not hand newly created projects to agents without Git-backed review history unless the user explicitly chooses that path. Missing Git should be recoverable at Execute time for older projects.

Unit 01 - Initialize Git During Onboarding

  • Status: complete
  • Ask to initialize Git when a project root is missing Git, defaulting to yes in non-interactive and in-app onboarding.
  • Create an initial scaffold commit after Hyperwiki generation when Git initialization is accepted.
  • Warn and offer Git initialization before Execute when an older project is missing Git.
