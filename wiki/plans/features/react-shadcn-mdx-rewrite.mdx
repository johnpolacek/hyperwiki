    <section className="panel">
      <h2>Decisions And Boundaries</h2>
      <div className="decision-grid">
        <article className="decision">
          <strong>Runtime boundary</strong>
          <p>Tauri and Rust remain the runtime owner. Node, Vite, Tailwind, shadcn, and MDX tooling are build/development surfaces, not a backend runtime.</p>
        </article>
        <article className="decision">
          <strong>Wiki format</strong>
          <p>All app-rendered wiki pages now live as <code>wiki/**/*.mdx</code>. Root agent files and skill files stay Markdown because external tools expect those conventions.</p>
        </article>
        <article className="decision">
          <strong>LLM surface</strong>
          <p>Agents receive Markdown-first derivatives and structured plan metadata, not raw rendered HTML, so visual MDX does not degrade context quality.</p>
        </article>
        <article className="decision">
          <strong>Planning flow</strong>
          <p><code>+ plan</code> opens a full-page agent interview. The agent uses <code>$hyperwiki</code> and grill-with-docs-style questioning, then writes staged MDX plans with verification for every executable unit.</p>
        </article>
      </div>
