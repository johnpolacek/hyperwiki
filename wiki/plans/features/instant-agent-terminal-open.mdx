<PlanSummary>
  <PlanSummaryItem label="Status" value="completed" />
  <PlanSummaryItem label="Scope" value="manual + agent startup latency" />
  <PlanSummaryItem label="Direction" value="hybrid TUI: terminal-first UI with hidden active-plan prewarm" />
</PlanSummary>

<PlanSection title="Intent">
Clicking <code>+ agent</code> should produce a visible terminal pane immediately. If a hidden general Codex session is already warm for the active plan scope, the click should promote it; otherwise the pane should appear optimistically while the backend starts the real terminal session.
</PlanSection>

<PlanSection title="Implementation">
- Render a pending visible agent pane before awaiting <code>/api/terminal/start</code>, using a client-generated terminal id that the backend accepts.
- Preserve pending panes during session reloads until the matching real session arrives or startup fails.
- Add one hidden standby <code>purpose="general"</code> agent prewarm for the active incomplete plan scope, separate from Modify prewarm.
- Promote matching hidden general sessions when <code>+ agent</code> is clicked.
- Add timing logs for click, pending pane insertion, backend start, first output, and prompt readiness.
</PlanSection>

<PlanSection title="Verification">
- <code>pnpm run check</code>
- Static smoke coverage for optimistic panes, pending preservation, and general-vs-modify prewarm separation.
</PlanSection>

<PlanSection title="Result">
Completed on 2026-06-06. Manual <code>+ agent</code> now promotes a hidden general agent when available, otherwise it inserts a pending visible pane immediately with a client-generated terminal id and reconciles that pane when the backend start returns. Session reloads preserve pending panes, failed optimistic starts stay dismissible, and timing milestones are logged.
</PlanSection>
