    <section>
      <h2>Intent</h2>
      <p>Replace the wterm-backed terminal renderer with xterm.js so terminal output keeps stable dimensions, handles PTY control sequences reliably, and survives collapse or expand without turning into a one-character column.</p>
    </section>
    <section>
      <h2>Implementation</h2>
      <ul>
        <li>Use <code>@xterm/xterm</code> with fit and web-link addons for browser terminal panels.</li>
        <li>Keep Hyperwiki's existing PTY websocket endpoint and session lifecycle, replacing only the browser transport adapter.</li>
        <li>Resize PTYs through the xterm fit addon instead of measuring character cells manually.</li>
        <li>Preserve collapse or expand position through xterm buffer viewport state.</li>
        <li>Allow the managed Run Dev terminal to start while collapsed and show a Starting state until the server prints a runtime URL.</li>
        <li>Use the runtime URL printed by the dev terminal for the Open App link when it differs from the configured preview URL.</li>
        <li>Serve xterm assets through the local vendor route and remove the obsolete wterm packages.</li>
      </ul>
    </section>
