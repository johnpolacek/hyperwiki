## Reader Goal

After 2 minutes, the reader can understand why staged plans must create child pages and how the implementation prevents monolithic staged plan files.

## Intent

The existing imported-MVP planner already requires separate stage and executable unit files, but the general new-plan workflow still allows a multi-stage plan to be written as one compact feature page. That breaks navigation and execution because feature pages are sidebar leaves.

<Scope>


- Tighten the general plan creation prompt so explicit multi-stage plans use a plan-root directory with stage and unit MDX pages.
- Add wiki validation warnings for plan pages that declare stages or current units but have no child stage/unit files.
- Keep compact feature plans valid when they do not claim a staged execution sequence.

</Scope>
