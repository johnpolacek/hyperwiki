<section>
  <h2>Intent</h2>
  <p>When a project dev server is already running, Hyperwiki should make the running terminal easy to reveal without making the same control also stop the process. Stop and Restart remain lifecycle actions; terminal reveal is a separate navigation action.</p>
</section>

<section>
  <h2>Implementation</h2>
  <ul>
    <li>Add a dedicated <code>dev terminal</code> control when preview reports a running dev server.</li>
    <li>Keep <code>stop dev</code> as a separate managed-process action.</li>
    <li>Add per-session collapsed terminal pane state with expandable headers and compact status summaries.</li>
    <li>Reveal, expand, select, and scroll to the dev pane when a live Hyperwiki-owned dev session exists.</li>
    <li>Keep detached managed dev sessions honest by showing that terminal output cannot be replayed after an app restart.</li>
  </ul>
</section>
