<section>
  <h2>Intent</h2>
  <p>Hyperwiki should present project dev state as one compact row: dev, running state, start or stop, PID when known, and collapse or expand. Dev should not add separate toolbar buttons, restart controls, close controls, or a competing empty terminal message.</p>
</section>

<section>
  <h2>Implementation</h2>
  <ul>
    <li>Render a single dev bar that shows <code>running</code> or <code>not running</code>, the managed PID when available, and one <code>start</code> or <code>stop</code> action.</li>
    <li>Remove separate <code>dev terminal</code>, <code>stop dev</code>, dev restart, dev close, and <code>No terminals running</code> UI.</li>
    <li>Keep per-session collapsed terminal pane state with expandable headers and compact status summaries.</li>
    <li>Reveal, expand, select, and scroll to the dev pane when a live Hyperwiki-owned dev session exists.</li>
    <li>Keep detached managed dev sessions honest by showing that terminal output cannot be replayed after an app restart.</li>
  </ul>
</section>
