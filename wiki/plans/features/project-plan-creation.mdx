    <section className="panel" aria-labelledby="qa-loop">
      <h2 id="qa-loop">Q&amp;A Clarification Loop</h2>
      <table className="matrix">
        <thead>
          <tr>
            <th>Gate</th>
            <th>LLM Behavior</th>
            <th>Exit Condition</th>
          </tr>
        </thead>
        <tbody>
          <tr>
            <td>Initial intake</td>
            <td>Ask for the feature outcome, user workflow, non-goals, affected surface, validation expectation, and hard constraints not visible from repo/wiki context.</td>
            <td>The LLM can summarize the intended plan in concrete terms and name remaining unknowns.</td>
          </tr>
          <tr>
            <td>Clarity scoring</td>
            <td>Classify open questions by impact: blocking, important, optional, or safely deferrable. Ask the next batch from the highest-impact unresolved category.</td>
            <td>No blocking questions remain, and important unknowns are answered or explicitly recorded as assumptions.</td>
          </tr>
          <tr>
            <td>Continuous follow-up</td>
            <td>After each answer, detect contradictions, newly exposed risks, missing acceptance criteria, schema/API/UX ambiguity, and validation gaps. Ask follow-ups instead of rushing to write.</td>
            <td>The plan has clear scope, non-goals, dependencies, implementation shape, first execution unit, and verification path.</td>
          </tr>
          <tr>
            <td>Write approval</td>
            <td>Present a compact proposed plan shape and ask for confirmation only when the remaining ambiguity would materially affect the plan.</td>
            <td>The user approves the shape, asks to skip questions, or all remaining unknowns are low-risk and documented.</td>
          </tr>
        </tbody>
      </table>
    </section>

        </li>
        <li>
          <h3>Unit 04 - Workspace UI</h3>
          <p>Add the project-scoped creation surface, Q&amp;A rail, path preview, dirty-state warning, conflict handling, clarity status, and create action from the planning workspace.</p>

        </li>

    <section className="decision-panel" aria-labelledby="decisions">
      <h2 id="decisions">Decisions And Assumptions</h2>
      <ul>
        <li><span className="badge status-active">Decision</span> Default to compact feature plans; promote to staged plans only after the user chooses a larger planning shape.</li>
        <li><span className="badge status-active">Decision</span> Creation should navigate to the new plan for review; it should not launch an agent implementation session automatically.</li>
        <li><span className="badge status-active">Decision</span> The LLM should keep asking clarification questions until no blocking unknowns remain and important unknowns are answered or explicitly recorded as assumptions.</li>
        <li><span className="badge status-decision">Assumption</span> The first implementation should support feature, fix, cleanup, and research labels as metadata without creating different file trees.</li>
        <li><span className="badge status-decision">Unknown</span> Whether pasted briefs should be captured as a source brief during creation or deferred to a later import flow.</li>
      </ul>
