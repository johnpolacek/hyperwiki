Dev Script Agent Guidance

Summary

  • Status: complete
  • Goal: make new-project and Execute handoffs explicitly require a runnable package dev script when building app previews.
  • Current unit: none
  • Validation: pnpm run check and pnpm run smoke:init.

Intent

Agents working on generated projects should not infer dev-preview setup from scattered docs. If the project is an app or previewable site, the handoff should explicitly tell the agent to create or update package.json with a runnable dev script, preferably backed by Portless.

Scope

  • Update new-project handoff prompts.
  • Update Execute prompts for main and worktree implementation.
  • Update generated AGENTS.md, development page, and technical brief defaults.
  • Keep existing Portless wrapping behavior for repositories that already have a dev script.

Outcome

  • New Project handoffs now tell agents to create or update package.json with a runnable dev script for previewable apps.
  • Execute handoffs now repeat the same requirement for main and worktree implementation.
  • Generated project guidance now names Portless-backed dev scripts and dev:app as the preferred structure.
