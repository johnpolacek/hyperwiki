<section>
  <h2>Intent</h2>
  <p>When Hyperwiki restarts while a project dev server it launched is still running, the next Run Dev action should not start a duplicate command that fails with an already-running server error. Hyperwiki should detect the managed process, show that dev is running, and offer stop or restart controls.</p>
</section>

<section>
  <h2>Implementation</h2>
  <ul>
    <li>Add managed dev preview metadata for running source, stop and restart capability, and persisted dev session process identity.</li>
    <li>Start dev through a dedicated lifecycle API with a deterministic project checkout dev session id.</li>
    <li>Keep external Portless routes observable but not stoppable unless they match a Hyperwiki-managed dev session.</li>
    <li>Render detached managed dev processes as lifecycle panes instead of attempting terminal replay after app restart.</li>
    <li>Stop managed dev processes through the recorded PTY process group when available, falling back to the recorded PID.</li>
  </ul>
</section>
