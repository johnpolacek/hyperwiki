<h1>Remove Legacy Node Runtime</h1>
    <section>
      <h2>Summary</h2>
      <ul>
        <li>Status: complete</li>
        <li>Shape: post-rewrite cleanup plan</li>
        <li>Current unit: Unit 01 - Tauri-only runtime contract complete</li>
        <li>Next action: choose or create the next product plan when new work begins.</li>
        <li>Validation: <code>pnpm run check:all</code>, <code>pnpm run tauri:build</code>, and Rust-backed <code>wt</code> CLI smoke paths passed.</li>
      </ul>
    </section>
    <section>
      <h2>Fixed Decisions</h2>
      <ul>
        <li><code>package.json</code> remains only for <code>pnpm</code> developer scripts, the Tauri CLI, frontend static checks, and asset smoke tests.</li>
        <li>The product command surface is the Rust/Tauri binary. No npm <code>bin</code> shim remains.</li>
        <li>Worktree commands stay convenient through <code>pnpm wt:*</code>, but those scripts invoke Rust CLI commands instead of a Node shim.</li>
        <li>The frontend uses Tauri command transport only. HTTP and WebSocket transports are no longer supported for hyperwiki's own app shell.</li>
      </ul>
    </section>
    <section>
      <h2>Units Of Work</h2>
      <ol>
        <li>Update package scripts and dependency ownership so <code>pnpm dev</code> launches Tauri and no legacy runtime packages remain.</li>
        <li>Delete the Node runtime and legacy smoke tests that imported it.</li>
        <li>Remove fetch/WebSocket frontend fallbacks and keep the static app assets mirrored.</li>
        <li>Add Rust CLI worktree subcommands for the repo-local worktree workflow.</li>
        <li>Replace current docs and wiki source truth that still describe Node as a live reference runtime.</li>
      </ol>
    </section>
