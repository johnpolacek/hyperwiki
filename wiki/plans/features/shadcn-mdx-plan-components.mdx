<PlanHero>
  <h1>shadcn MDX Plan Components</h1>
  <p>Bring a Holocron-inspired MDX component vocabulary into Hyperwiki plans without importing Holocron runtime dependencies. Components stay Hyperwiki-owned, render through the trusted local MDX renderer, and use shadcn/ui primitives where they improve plan readability and design quality.</p>
</PlanHero>

<Decision title="Runtime boundary">
  <p>Do not depend on Holocron, Spiceflow, hosted assistants, icon atlases, Mermaid, Prism virtual modules, or public docs deployment conventions. Hyperwiki only adopts the useful component ideas.</p>
</Decision>

<Decision title="Component boundary">
  <p>Use actual shadcn/ui primitives for cards, badges, alerts, tabs, accordions, separators, scroll areas, collapsibles, tooltips, and buttons. Use shadcn-style local wrappers for plan-specific concepts such as decisions, evidence, verification, steps, fields, trees, prompts, updates, and visibility.</p>
</Decision>

<Decision title="Agent context boundary">
  <p>Visual MDX must not degrade agent context. App-only human content can be wrapped in <code>Visibility for="humans"</code>, while agent-only context can be wrapped in <code>Visibility for="agents"</code>. Markdown derivatives should strip human-only content and preserve agent-only content.</p>
</Decision>
