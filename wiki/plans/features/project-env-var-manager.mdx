<PlanSection title="Intent">
Users should be able to satisfy manual gates such as Clerk or Convex key setup from inside Hyperwiki without opening a code editor. The easiest compatible path is editing the active checkout's <code>.env.local</code>, but secret values must not pass through terminal input, session replay, agent prompts, or logs.
</PlanSection>

<PlanSection title="Scope">
- Add a Rust-owned project env API that reads <code>.env.example</code>, safely updates <code>.env.local</code>, masks values in responses, and blocks unsafe writes when <code>.env.local</code> is not ignored.
- Add a terminal header <code>env</code> action and terminal-output missing-key detection that opens a secure Hyperwiki env sheet instead of sending secrets through the PTY.
- Add a Project Settings fallback using the same env editor.
- Keep v1 file-based only; do not add a separate encrypted vault or runtime env injection layer.
</PlanSection>

<PlanSection title="Non-Goals">
- Hyperwiki will not commit <code>.env.local</code>.
- Hyperwiki will not manage provider dashboards or hosted-service secrets.
- Hyperwiki will not type secret values into live terminals.
</PlanSection>

<PlanSection title="Unit 01 - Safe terminal-friendly env setup">
Implemented the backend API, UI sheet, missing-key affordance, settings fallback, and tests in one pass. The UI shows the active checkout path, masks values by default, offers reveal per field, creates or updates <code>.env.local</code>, and provides rerun guidance after save.
</PlanSection>

<PlanSection title="Follow-up - Non-modal drawer">
On 2026-06-07, the Project Env editor changed from a centered modal with a full-screen backdrop to a bottom-left non-modal drawer. This keeps terminal instructions visible while users paste Clerk, Convex, or other manual-gate values into the env editor.
</PlanSection>

<PlanSection title="Follow-up - Stable field layout">
On 2026-06-07, the drawer's env rows changed to a stacked key/value layout. Long keys no longer compete with the value input in a narrow horizontal grid, and the reveal button stays in a fixed column beside the value field.
</PlanSection>
