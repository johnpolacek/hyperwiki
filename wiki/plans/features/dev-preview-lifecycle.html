Intent

Make the App control describe dev server lifecycle state instead of collapsing missing dev commands into the misleading App Not Configured label, and recover when a project gains a package dev script after Hyperwiki onboarding.

Implementation

  • Keep App Not Configured only for projects without a preview URL.
  • Show Dev Not Available when a preview URL exists but no dev command is configured.
  • Show Run Dev when Hyperwiki can start the configured dev command.
  • Derive the dev command from package.json when .hyperwiki/config.json still has an empty command.
  • Add a derived dev panel before the CLI panel when the layout lacks one but the package dev script exists.
  • Show Stop Dev only for a running preview backed by the current Hyperwiki-managed dev terminal.
  • Show Dev Running for externally started Portless routes that Hyperwiki should not stop.

Verification

  • Add smoke coverage for preview URL plus empty dev command.
  • Update browser smoke expectations for the lifecycle label.
  • pnpm run check passed.
  • pnpm run smoke:app-preview passed with coverage for blank config command falling back to package.json.
  • Browser smoke was rerun after the plan dashboard returned to no active unit.
