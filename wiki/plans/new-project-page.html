New Project Page

Summary

  • Status: complete
  • Goal: move New Project out of the Manage Projects view into its own app-shell page.

Acceptance

  • The topbar Projects dropdown links to both Manage Projects and New Project.
  • /projects shows project management without the project creation form.
  • /projects/new shows the project creation form as the primary page.
  • Direct refresh on /projects/new opens the New Project page.
  • Successful project creation returns to Manage Projects and starts the agent handoff.

Result

  • Added /projects/new as a dedicated app-shell page for project creation.
  • Moved the project brief form out of Manage Projects.
  • Added New Project and Manage Projects actions to the Projects dropdown.
  • Removed wiki command bar and terminal chrome from project-management app pages so the New Project form fills the workspace cleanly.
  • Redesigned the MVP planning interview into a lighter decision console with a compact evidence rail, visible right-side step count, calmer option cards, accent-backed selection, mobile-first decision ordering, wider header-aligned layout, reduced mono overuse, and a unified footer.
  • Verified direct refresh and browser workflow coverage for /projects/new.
