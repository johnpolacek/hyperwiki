Non-plan Wiki Prompt Rail

Summary

  • Status: complete
  • Shape: focused UI workflow fix
  • Current unit: implement non-plan wiki prompt rail
  • Next action: monitor dogfooding for any remaining non-plan page command bar confusion.
  • Validation: pnpm run check, pnpm run smoke:browser, pnpm run smoke:ideas, and targeted agent-browser review.

Intent

Non-plan wiki pages should keep the reading surface calm. The right side should invite a page-specific agent edit only when the user asks for it, instead of opening terminal panels automatically.

Behavior

  • On non-plan wiki pages, show a right-side prompt labeled Modify Idea.
  • Do not show terminal creation controls on non-plan wiki pages.
  • Submitting the prompt replaces the prompt rail with one full-height agent terminal and forwards a revise-this-idea-page instruction.
  • Plan pages keep the existing plan-prompt and terminal workflow.

Outcome

  • Implemented: imported idea pages no longer auto-open agent and dev terminals.
  • Implemented: non-plan wiki pages show a right-side Modify Idea rail until the user submits a prompt.
  • Implemented: submitted non-plan prompts open exactly one full-height agent terminal without manual New Agent or New CLI controls.
  • Implemented: Up Next is hidden on non-plan wiki pages because there is no active plan context.
