Stage 05 - Multi-project Workspace

Summary

  • Status: complete
  • Current unit: none
  • Next action: none; this stage is complete.
  • Blockers: none
  • Validation: pnpm run check, pnpm run smoke:launch, pnpm run smoke:browser, pnpm run smoke:init, pnpm run smoke:sessions, pnpm run smoke:pty.

Intent

Let hyperwiki manage multiple local projects while preserving the existing single-project feel until more than one project exists.

Execution Units

  1. Unit 01 - Registry and Launch Foundation - complete
  2. Unit 02 - Project-scoped Server Runtime - complete
  3. Unit 03 - Multi-project UI - complete

Unit 04 - Worktree-aware Pretty URLs

  • Status: complete
  • Keep Git worktrees as separate swappable project views instead of deduplicating them by common repository.
  • Add readable project and worktree slugs so launch and switching can use URLs such as /workspace/hyperwiki/main and /workspace/hyperwiki/plan-unit-navigation.
  • Use root-based project IDs as the runtime source of truth while preserving legacy ?project=<id> URLs.
  • Implemented with project/worktree slug routing, worktree labels in the project switcher, and legacy project ID URL compatibility.
  • Verification: pnpm run check, pnpm run smoke:launch, and isolated browser smoke with HYPERWIKI_SMOKE_URL=http://127.0.0.1:4317/workspace/hyperwiki/main pnpm run smoke:browser.

Unit 05 - Missing Worktree Pruning

  • Status: complete
  • Automatically remove registered non-main worktree entries when their root path no longer exists.
  • Keep missing main or standalone project entries visible as unavailable so users can diagnose or restore them.
  • Changed the Projects panel to show distinct projects only; worktree context moved to the terminal toolbar.
  • Verification: pnpm run check and pnpm run smoke:launch.

Unit 06 - Faster Project Switching

  • Status: complete
  • Make project dropdown switches feel immediate by updating the selected project, route, and visible wiki page before secondary project metadata refreshes finish.
  • Keep current-work resolution, sidebar refresh, repository context, workspace summary, guardrails, and terminal restoration in the background with stale-switch protection.
  • Verification: pnpm run check and pnpm run smoke:tauri-static-assets.

Non-goals

  • No folder scanning or arbitrary non-hyperwiki project registration in v1.
  • No browser add-project flow in v1.
  • No shared cross-project terminal runtime.
