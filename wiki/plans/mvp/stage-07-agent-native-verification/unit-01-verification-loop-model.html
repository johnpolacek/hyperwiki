Unit 01 - Verification Loop Model

Stage 07 - Agent-native Verification

Summary

  • Status: active
  • Define the repo-local shape for verification loops: command, scope, trigger, status, last run, evidence, and whether the result is automated or manual.
  • Expose verification loops through a local API that can be consumed by the browser workspace and future MCP tools.
  • Keep recorded verification state separate from canonical claims unless a human or agent intentionally records the outcome into the wiki.
  • Verification: pnpm run check plus targeted API smoke coverage for reading configured verification loops.
