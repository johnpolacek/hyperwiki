<h2>Intent</h2>
<p>Let hyperwiki manage multiple local projects while preserving the existing single-project feel until more than one project exists.</p>

  <h2>Execution Units</h2>
  <ol>
    <li><a href="/wiki/plans/mvp/stage-05-multi-project/unit-01-registry-and-launch-foundation.mdx">Unit 01 - Registry and Launch Foundation</a> - complete</li>

<section>
  <h3>Unit 04 - Worktree-aware Pretty URLs</h3>
  <ul>
    <li>Status: complete</li>
    <li>Keep Git worktrees as separate swappable project views instead of deduplicating them by common repository.</li>
    <li>Add readable project and worktree slugs so launch and switching can use URLs such as <code>/workspace/hyperwiki/main</code> and <code>/workspace/hyperwiki/plan-unit-navigation</code>.</li>
    <li>Use root-based project IDs as the runtime source of truth while preserving legacy <code>?project=&lt;id&gt;</code> URLs.</li>
    <li>Implemented with project/worktree slug routing, worktree labels in the project switcher, and legacy project ID URL compatibility.</li>
    <li>Verification: <code>pnpm run check</code>, <code>pnpm run smoke:launch</code>, and isolated browser smoke with <code>HYPERWIKI_SMOKE_URL=http://127.0.0.1:4317/workspace/hyperwiki/main pnpm run smoke:browser</code>.</li>
  </ul>
</section>
<section>
  <h3>Unit 05 - Missing Worktree Pruning</h3>
  <ul>
    <li>Status: complete</li>
    <li>Automatically remove registered non-<code>main</code> worktree entries when their root path no longer exists.</li>
    <li>Keep missing <code>main</code> or standalone project entries visible as unavailable so users can diagnose or restore them.</li>
    <li>Changed the Projects panel to show distinct projects only; worktree context moved to the terminal toolbar.</li>
    <li>Verification: <code>pnpm run check</code> and <code>pnpm run smoke:launch</code>.</li>
  </ul>
</section>
<section>
  <h3>Unit 06 - Faster Project Switching</h3>
  <ul>
    <li>Status: complete</li>
    <li>Make project dropdown switches feel immediate by updating the selected project, route, and visible wiki page before secondary project metadata refreshes finish.</li>
    <li>Keep current-work resolution, sidebar refresh, repository context, workspace summary, guardrails, and terminal restoration in the background with stale-switch protection.</li>
    <li>Verification: <code>pnpm run check</code> and <code>pnpm run smoke:tauri-static-assets</code>.</li>
  </ul>
</section>
<h2>Non-goals</h2>
<ul>
  <li>No folder scanning or arbitrary non-hyperwiki project registration in v1.</li>
  <li>No browser add-project flow in v1.</li>
  <li>No shared cross-project terminal runtime.</li>
</ul>
