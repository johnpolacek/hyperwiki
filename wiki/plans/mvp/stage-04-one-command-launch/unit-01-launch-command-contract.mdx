<h1>Unit 01 - Launch Command Contract</h1>
<p><a href="/wiki/plans/mvp/stage-04-one-command-launch.mdx">Stage 04 - One-command Local Launch</a></p>
<section className="summary">
  <h2>Summary</h2>
<ul>
    <li>Status: complete</li>
    <li>Define the public command surface for starting hyperwiki as a complete local workspace. Favor the existing CLI and package scripts over introducing a separate launcher.</li>
    <li>The command should start the local server, wait until the workspace is reachable, open the browser to the workspace URL, and ensure the default wterm operating session is created or visible.</li>
    <li>Keep the command local-first and repo-root aware. It must preserve ignored runtime state under <code>.hyperwiki/</code> and avoid writing terminal runtime data into canonical wiki files.</li>
    <li>Acceptance: a user can run one documented command from the repository root and immediately land in a browser workspace with the default terminal workflow available.</li>
    <li>Implemented: <code>hyperwiki launch</code> and bare <code>hyperwiki</code> start or attach to the local server, wait for workspace readiness, open the browser workspace, and print the workspace URL.</li>
  </ul>
</section>
