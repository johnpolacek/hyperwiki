<section className="summary">
  <h2>Summary</h2>
  <ul>
    <li>Status: complete</li>
    <li>Shape: focused UI workflow fix</li>

  </ul>
</section>
<section>
  <h2>Intent</h2>
  <p>Non-plan wiki pages should keep the reading surface calm. The right side should invite a page-specific agent edit only when the user asks for it, instead of opening terminal panels automatically.</p>
</section>
<section>
  <h2>Behavior</h2>
  <ul>
    <li>On non-plan wiki pages, show a right-side prompt labeled <code>Modify Idea</code>.</li>
    <li>Do not show terminal creation controls on non-plan wiki pages.</li>
    <li>Submitting the prompt replaces the prompt rail with one full-height agent terminal and forwards a revise-this-idea-page instruction.</li>
    <li>Plan pages keep the existing plan-prompt and terminal workflow.</li>
  </ul>
</section>
<section>
  <h2>Outcome</h2>
  <ul>
    <li>Implemented: imported idea pages no longer auto-open agent and dev terminals.</li>
    <li>Implemented: non-plan wiki pages show a right-side <code>Modify Idea</code> rail until the user submits a prompt.</li>
    <li>Implemented: submitted non-plan prompts open exactly one full-height agent terminal without manual New Agent or New CLI controls.</li>
    <li>Implemented: Up Next is hidden on non-plan wiki pages because there is no active plan context.</li>
  </ul>
</section>
