<section className="summary">
  <h2>Summary</h2>
<ul>
    <li>Status: complete</li>
    <li>Update product copy and workspace labels from a terminal/wiki workspace toward a repo-local agent operating layer.</li>
    <li>Keep the UI utilitarian: surface current context, verification loops, guardrails, and review workflows without creating a marketing page.</li>
    <li>Current slice: make Dashboard a literal workspace page for Ideas and Projects, with create/import flows that hand a markdown or HTML brief to the active agent.</li>
    <li>Dashboard behavior: hide terminal panels until a create/import action starts an agent on the right; idea and project generation remain agent-driven from user-provided documents.</li>
    <li>Fresh project behavior: initialize a new hyperwiki project, switch the workspace into it, then start the agent with the imported document and page-generation instructions.</li>
    <li>Implemented: Dashboard is available at <code>/dashboard</code>, lists ideas and projects, supports document import forms, initializes new projects through <code>/api/projects/create</code>, and hands idea/project markdown or HTML to an agent terminal.</li>
    <li>Follow-up slice: idea and project imports accept HTML files, infer titles from document headings or titles, show a clear manual-or-import separator, and auto-start the agent after a file is selected.</li>
    <li>Verification: covered by <code>pnpm run check</code>, <code>pnpm run smoke:ideas</code>, and browser smoke coverage for Dashboard mode, forms, document import affordances, command bar behavior, and agent handoff visibility.</li>
  </ul>
