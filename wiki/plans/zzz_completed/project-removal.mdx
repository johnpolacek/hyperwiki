<h1>Project Removal</h1>
    <section className="summary">
      <h2>Summary</h2>
      <ul>
        <li>Status: complete</li>
        <li>Shape: compact feature plan</li>
        <li>Current unit: Unit 01 - Remove Project From Manage Projects</li>
        <li>Next action: monitor dogfooding feedback for remove/delete wording and deletion safety.</li>
        <li>Blockers: none</li>
      </ul>
    </section>
    <section>
      <h2>Intent</h2>
      <p>Let users remove stale or unwanted projects from hyperwiki's registry from the Manage Projects page, with an explicit opt-in path for deleting the underlying project files.</p>
    </section>
    <Scope>
      <ul>
        <li>Add a small remove action at the bottom-right of each project card.</li>
        <li>Use inline confirmation on the card, including a destructive warning box.</li>
        <li>Default to registry-only removal; leave project files untouched unless the user checks an explicit delete-files checkbox.</li>
        <li>Hide the wiki sidebar on Projects, New Project, and Settings so management pages feel global rather than bound to a current wiki page.</li>
      </ul>
    </Scope>
    <section>
      <h2>Non-goals</h2>
      <ul>
        <li>No undo or trash/archive workflow.</li>
        <li>No bulk project management.</li>
        <li>No deletion of active terminal sessions beyond registry/file cleanup behavior already implied by project removal.</li>
      </ul>
    </section>
    <section>
      <h2>Unit 01 - Remove Project From Manage Projects</h2>
      <ul>
        <li>Add server registry removal and optional recursive project-root deletion.</li>
        <li>Add inline confirmation UI and reload the project lists after removal.</li>
        <li>Add smoke coverage for registry-only removal, opt-in file deletion, and management-page sidebar hiding.</li>
      </ul>
