<section>
  <h2>Intent</h2>
  <p>Users should be able to switch the sidebar from Plans to Bugs, report product defects quickly, and ask an agent to fix a specific bug without converting it into a plan first. Bugs are durable project knowledge, not transient screenshot feedback.</p>
</section>

<section>
  <h2>Decisions</h2>
  <ul>
    <li><strong>Storage:</strong> bug reports are repo-visible MDX files under <code>wiki/bugs/</code> with <code>wikiKind: "bug"</code>.</li>
    <li><strong>Creation:</strong> <code>+ Bug</code> opens a compact form for title, severity, repro context, expected/actual behavior, current wiki route, active checkout context, and optional linked plan or unit.</li>
    <li><strong>Lifecycle:</strong> v1 statuses are <code>open</code>, <code>fixing</code>, <code>fixed</code>, <code>verified</code>, and <code>closed</code>.</li>
    <li><strong>Fixing:</strong> bug pages can run a direct bug-scoped agent fix using <code>scopeKind: "bug"</code>, separate from plan-scoped terminals and screenshot review gates.</li>
  </ul>
</section>

<section>
  <h2>Implementation</h2>
  <ol>
    <li>Add a Rust bug domain with list, create, and status-update endpoints that read and write guarded <code>wiki/bugs/**/*.mdx</code> paths.</li>
    <li>Extend wiki metadata/status parsing so bug statuses are recognized without changing plan completion derivation.</li>
    <li>Add frontend bug types, API helpers, sidebar mode switching, bug list rendering, empty state, and quick report dialog.</li>
    <li>Add bug route behavior: header actions for Fix Bug, mark verified, close, and reopen; bug pages use bug-specific terminal scope.</li>
    <li>Add open bug summaries to project contract and MCP resource context so agents can see outstanding defects.</li>
  </ol>
</section>
