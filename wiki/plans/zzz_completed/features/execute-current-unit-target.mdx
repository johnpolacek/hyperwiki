<section>
  <h2>Intent</h2>
  <p>Execute should always run the current unit. A user can inspect or modify another plan page, but the Execute button must not silently do nothing merely because that visible page is not the current work item.</p>
</section>
<section>
  <h2>Implementation</h2>
  <ul>
    <li>Resolve Execute prompts from the current work path already computed for the plan tree.</li>
    <li>Scope Execute terminals to that current unit's plan root instead of the visible stale page.</li>
    <li>Start a fresh general agent session for Execute while preserving Modify's prewarmed <code>purpose="modify"</code> session behavior.</li>
    <li>Keep Execute disabled only when there is no resolvable current work page or the resolved current page is complete.</li>
  </ul>
</section>
