## Intent

hyperwiki should keep its quiet local-devtool identity while reading less like a document page on operational surfaces. This pass preserves IA and behavior, then improves app typography, form hierarchy, CTA contrast, and token consistency.

<ImplementationNotes>


- Split app UI typography from document typography so shell chrome, forms, settings, and project controls use a sans UI voice while rendered docs can keep their reader font.
- Reduce heavy onboarding card treatment and improve disabled CTA readability.
- Keep lucide icons because the project shadcn configuration selects lucide.
- Route terminal and status color choices through existing semantic tokens where practical, without changing terminal behavior.

</ImplementationNotes>
