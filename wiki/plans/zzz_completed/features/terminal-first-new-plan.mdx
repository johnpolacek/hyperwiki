<section>
  <h2>Intent</h2>
  <p>Regular <code>+ plan</code> should open a visible planning agent terminal. The agent should ask Q&amp;A in the terminal until the work is decision-complete, then write durable MDX plan docs.</p>
</section>

<section>
  <h2>Implementation</h2>
  <ul>
    <li>Remove the regular full-page plan creation route and keep the workspace shell visible.</li>
    <li>Make <code>+ plan</code> navigate to the Plans index and start a fresh visible agent terminal scoped to that page.</li>
    <li>Update the prompt so Q&amp;A happens in terminal prose instead of the retired structured app-rendered JSON question flow.</li>
    <li>Leave imported-project onboarding Q&amp;A unchanged.</li>
  </ul>
</section>

<section>
  <h2>2026-06-03 Regression Fix</h2>

    <li>The blank-intent prompt explicitly forbids repo inspection, wiki reads, command execution, and existing-plan summaries before that first answer.</li>
    <li>After the user supplies a focus, the normal evidence-first planning workflow resumes.</li>
