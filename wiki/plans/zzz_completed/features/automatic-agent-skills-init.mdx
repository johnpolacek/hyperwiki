    <section>
      <h2>Reader Goal</h2>
      <p>After 2 minutes, a maintainer can implement deterministic repo-local skill installation for every new hyperwiki project without adding network-dependent init behavior.</p>
    </section>
    <section>
      <h2>Decision</h2>
      <ul>
        <li>Install agent skills by default for <code>hyperwiki init</code> and dashboard-created projects.</li>
        <li>Use a vendored, pinned bundle compiled into the Rust binary so init works offline and in packaged builds.</li>
        <li>Expose <code>hyperwiki init --no-skills</code> for users who want a minimal scaffold.</li>
        <li>Install these skills initially: <code>hyperwiki</code>, <code>parallel-dev-worktrees</code>, <code>portless</code>, <code>frontend-design</code>, <code>shadcn</code>, and <code>tailwind-design-system</code>.</li>
        <li>Follow-up expansion: add <code>grill-with-docs</code> for plan/domain-language stress tests and <code>make-interfaces-feel-better</code> for UI polish guidance.</li>
      </ul>
    </section>
    <section>
      <h2>Implementation</h2>
      <ol>
        <li><strong>Bundle source.</strong> Add curated skill snapshots under a repo-owned source directory, excluding temp files, screenshots, caches, and system artifacts.</li>
        <li><strong>Build embed.</strong> Extend <code>src-tauri/build.rs</code> to generate Rust constants from the vendored skill files and include those bytes in the binary.</li>
        <li><strong>Installer.</strong> Add Rust project-domain helpers that write <code>.agents/skills/&lt;skill&gt;/...</code> and merge <code>skills-lock.json</code> without overwriting unrelated user entries.</li>
        <li><strong>Init surface.</strong> Add <code>install_agent_skills</code> to init options, default it on for CLI and dashboard paths, and wire <code>--no-skills</code> into the CLI.</li>
        <li><strong>Guidance.</strong> Update generated <code>AGENTS.md</code>, <code>wiki/AGENTS.mdx</code>, README, and source docs to describe the default repo-local skills.</li>
      </ol>
    </section>
