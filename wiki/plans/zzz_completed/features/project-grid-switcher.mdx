<section>
  <h2>Intent</h2>
  <p>The topbar <strong>Projects</strong> action should open the full Projects grid instead of a small popover. The grid remains the compact, work-focused management surface, and clicking a project card switches to that project through the existing project hydration flow.</p>
</section>

<section>
  <h2>Implementation</h2>
  <ul>
    <li>Changed the topbar Projects button to navigate directly to <code>/projects</code>.</li>
    <li>Removed the topbar project-switch popover and its now-unused state/props.</li>
    <li>Made each project card a keyboard-accessible switch target using the existing <code>onOpenProject</code> flow.</li>
    <li>Stopped propagation from open, remove, cancel, confirm, and delete-files controls so nested management actions do not double-trigger or switch unexpectedly.</li>
    <li>Follow-up polish removed the page-level grid-beam background from the Projects grid and added immediate card opening feedback with a busy state, primary ring, spinner label, and disabled nested controls while project data hydrates.</li>
    <li>Added static coverage for the route trigger, card-level switch affordance, keyboard activation, and removal controls.</li>
  </ul>
</section>
