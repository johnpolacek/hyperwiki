    <section>
      <h2>Intent</h2>
      <p>Make the workspace sidebar navigable with arrow keys so Plans, nested plan groups, and the Project group can be traversed without tabbing through every link.</p>
    </section>
    <section>
      <h2>Scope</h2>
      <ul>
        <li>Add roving focus across visible sidebar rows.</li>
        <li>Use tree-style keys: Up, Down, Left, Right, Home, End, Enter, and Space.</li>
        <li>Keep activation explicit: arrowing moves focus or expands groups, while Enter, Space, and click activate links.</li>
        <li>Preserve current sidebar layout, active-page styling, completion styling, and plan tree open-state behavior.</li>
      </ul>
    </section>
    <section>
      <h2>Unit 01 - Roving Sidebar Focus</h2>
      <ul>
        <li>Install keyboard handling after the dynamic sidebar renders from <code>/api/wiki</code>.</li>
        <li>Keep one visible sidebar row in the Tab order and move focus internally with arrow keys.</li>
        <li>Expose expandable row state with <code>aria-expanded</code> while preserving native <code>details</code> behavior.</li>
        <li>Add browser smoke coverage for traversal, expand/collapse, activation, and Project group participation.</li>
      </ul>
