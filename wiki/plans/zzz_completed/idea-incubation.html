Idea Incubation

Summary

  • Status: complete
  • Shape: compact feature plan
  • Current unit: implement ideas dashboard, wiki scaffold, and project promotion
  • Next action: dogfood the dashboard with real ideas and refine creation ergonomics after use.
  • Blockers: none
  • Validation: pnpm run check, pnpm run smoke:init, pnpm run smoke:ideas, pnpm run smoke:launch, and browser workspace smoke against an isolated dry-run server.

Intent

Add ideas as a lightweight incubation layer for concepts that are not projects yet. Ideas should stay free-form and repo-visible until the user promotes one into a real hyperwiki project.

Scope

  • Create a top-level wiki/ideas/ area with an index page and workspace navigation.
  • Add a hyperwiki Dashboard that shows ideas and registered projects together.
  • Add an explicit, confirmed action that initializes an idea as a new local hyperwiki project under ~/Projects.
  • Seed promoted projects with a planning package from the idea and replace the original idea with a compact promoted note.

Non-goals

  • No structured idea template, status lifecycle, tags, remote repository creation, or generated application code in this unit.
  • No hidden idea storage under ignored runtime state.

Verification

  • Verified syntax checks, generated idea scaffold, idea listing, path rejection, promotion, project seeding, origin replacement, and one-command launch.
  • Promotion uses a unique project root under ~/Projects by default and uses HYPERWIKI_PROJECTS_DIR in smoke coverage.
