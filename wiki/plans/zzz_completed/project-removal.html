Project Removal

Summary

  • Status: complete
  • Shape: compact feature plan
  • Current unit: Unit 01 - Remove Project From Manage Projects
  • Next action: monitor dogfooding feedback for remove/delete wording and deletion safety.
  • Blockers: none

Intent

Let users remove stale or unwanted projects from Hyperwiki's registry from the Manage Projects page, with an explicit opt-in path for deleting the underlying project files.

Scope

  • Add a small remove action at the bottom-right of each project card.
  • Use inline confirmation on the card, including a destructive warning box.
  • Default to registry-only removal; leave project files untouched unless the user checks an explicit delete-files checkbox.
  • Hide the wiki sidebar on Projects, New Project, and Settings so management pages feel global rather than bound to a current wiki page.

Non-goals

  • No undo or trash/archive workflow.
  • No bulk project management.
  • No deletion of active terminal sessions beyond registry/file cleanup behavior already implied by project removal.

Unit 01 - Remove Project From Manage Projects

  • Add server registry removal and optional recursive project-root deletion.
  • Add inline confirmation UI and reload the project lists after removal.
  • Add smoke coverage for registry-only removal, opt-in file deletion, and management-page sidebar hiding.
