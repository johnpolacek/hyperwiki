Ideas and Projects Route Split

Summary

  • Status: complete
  • Shape: compact feature plan
  • Current unit: complete
  • Next action: archive this plan during the next planning cleanup.
  • Blockers: none
  • Validation: pnpm run check and targeted browser route checks passed.

Intent

Dashboard has become an Ideas surface. Projects should move to a dedicated workspace page that is reachable from the Projects dropdown, while the top navigation should expose Ideas directly at /ideas.

Scope

  • Change the topbar Dashboard button into an Ideas button that navigates to /ideas.
  • Move project list and new-project import form into a dedicated /projects workspace page.
  • Add a Manage Projects link to the Projects dropdown.
  • Keep existing idea create, open, sidebar refresh, and promote behavior.
  • Keep existing project switching and project creation behavior, but relocate project management UI.

Implementation Result

  • The topbar Dashboard control is now an Ideas button that opens /ideas.
  • The Ideas page contains only idea list/import behavior.
  • Projects moved to /projects with the project list and new-project import form.
  • The Projects dropdown now includes a Manage Projects link.
  • Legacy /dashboard requests are routed to the Ideas surface for compatibility.

Non-Goals

  • No registry schema changes.
  • No new persisted project metadata.
  • No change to promoted project creation semantics.
