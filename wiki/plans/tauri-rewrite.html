Complete Tauri Rewrite

Rewrite Hyperwiki as a dedicated Tauri desktop app with a Rust-owned runtime, feature compatibility with the current Node/localhost product, full compatibility for existing wiki/project data, and permissioned local tooling that keeps the developer machine as the trust boundary.

Reader Goal

After 2 minutes, the reader should understand the migration shape, feature-parity gates, current execution unit, and why this is a Rust-core rewrite instead of a Tauri shell around the Node server.

Fixed Decisions

Rewrite boundary

Full Tauri rewrite. Rust owns backend/runtime behavior; Node is not the long-term product runtime.

Release target

Feature compatibility with the current Hyperwiki product, not a thin proof of concept.

Compatibility

Full compatibility with existing repo wiki files, .hyperwiki state shape, project registry expectations, Portless/worktree workflows, and current CLI habits.

Frontend

Keep the current vanilla TypeScript/HTML/CSS shape unless a staged Tailwind or shadcn migration reduces implementation risk. Do not combine framework churn with parity-critical Rust work by default.

Continuity

The rewrite branch may go dark. The plan still preserves the existing Node implementation as a reference until parity is proven.

Security posture

Use Tauri capabilities and Rust commands as the default boundary. Avoid making shell sidecars the main product architecture.

Risks And Defaults

PTY parity

Rust PTY behavior is the highest-risk subsystem. Prove detach/reattach, resize, export, and fallback behavior before porting review/MCP layers.

Testing gap

Tauri desktop WebDriver support is not equivalent to current browser automation on macOS. Use Rust tests, fixture snapshots, targeted app checks, and explicit manual desktop evidence.

Compatibility drift

Treat current tests and technical brief as the source of parity, then update the matrix when implementation discovers drift.

Frontend churn

Tailwind or shadcn may help later, but the default is no framework migration until the Tauri command contract and terminal runtime are stable.

Whole-Plan Acceptance

  • Hyperwiki ships as a Tauri desktop app with no browser chrome and a Rust-owned local runtime.
  • Existing initialized projects, wiki files, settings, project registry records, worktree conventions, and Portless preview patterns remain usable.
  • The app reaches feature compatibility across Projects, New Project, wiki navigation, plan command bar, Execute, Modify, terminals, worktrees, Settings, verification, review workflows, MCP, and app previews.
  • Node server/runtime dependencies are removed from the shipping path after parity closeout.
  • Durable docs and wiki pages identify Tauri/Rust as the canonical architecture.
