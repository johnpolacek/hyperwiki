<section>
  <h2>Plan Reconciliation Policy</h2>
  <ul>
    <li>If out-of-order work still belongs to the same sequence, renumber the active plan before continuing so unit order matches intended execution.</li>
    <li>If out-of-order work is a separate durable track, split it into its own active or completed plan and update the dashboard to keep exactly one current unit.</li>
    <li>Workspace current-state indicators should be derived from wiki summary status, not hardcoded to a stage or unit path.</li>
  </ul>
