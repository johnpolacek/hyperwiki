## Phase goal

A coherent design foundation: tokens (color, spacing, type, radius, dark mode) first, then a customized shadcn/ui component set consuming those tokens — not default shadcn.

## Entry gate

Phase 1 cleared (purpose and user stories settled).

## Sub-agent contract

- **Skills**: load `tailwind-design-system` then `shadcn` before changing tokens or components.
- **Produces**: design tokens, a customized shadcn/ui layer, and the `/wiki/plans/design-system/` sub-plan documenting the token system and component inventory with `Screen`-level specs where components are visual.

<Verification>

- The token system is defined once and consumed by components (no hardcoded values).
- The `design-system` sub-plan is complete and its units verified.

</Verification>
