    <section aria-labelledby="fixed-decisions">
      <h2 id="fixed-decisions">Fixed Decisions</h2>
      <div className="summary-grid">
        <article className="decision"><h3>Rewrite boundary</h3><p>Full Tauri rewrite. Rust owns backend/runtime behavior; Node is not the long-term product runtime.</p></article>
        <article className="decision"><h3>Release target</h3><p>Feature compatibility with the current hyperwiki product, not a thin proof of concept.</p></article>
        <article className="decision"><h3>Compatibility</h3><p>Full compatibility with existing repo wiki files, <code>.hyperwiki</code> state shape, project registry expectations, Portless/worktree workflows, and current CLI habits.</p></article>
        <article className="decision"><h3>Frontend</h3><p>Keep the current vanilla TypeScript/HTML/CSS shape unless a staged Tailwind or shadcn migration reduces implementation risk. Do not combine framework churn with parity-critical Rust work by default.</p></article>
        <article className="decision"><h3>Continuity</h3><p>The rewrite branch may go dark. The plan still preserves the existing Node implementation as a reference until parity is proven.</p></article>
        <article className="decision"><h3>Security posture</h3><p>Use Tauri capabilities and Rust commands as the default boundary. Avoid making shell sidecars the main product architecture.</p></article>
