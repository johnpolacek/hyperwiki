Portless Worktree Runtime

Summary

  • Status: completed
  • Goal: make local previews use Portless by default and add repo-local commands for parallel worktree development.
  • Preview pattern: https://hyperwiki.localhost on main and https://<branch-slug>.hyperwiki.localhost in feature worktrees.
  • Finish policy: merge feature branches into the integration branch to preserve history.

Implementation

  • Move the real app dev command behind a Portless-backed pnpm dev.
  • Make the hyperwiki dev server honor Portless-assigned app ports while keeping localhost-only binding.
  • Add worktree commands for doctor, create, list, resume, open, finish, and prune flows.
  • Document Portless and worktree usage in repo guidance and durable wiki pages.

Validation

  • pnpm run check passed.
  • pnpm wt:doctor, pnpm wt:list, and dry-run pnpm wt:open main passed.
  • pnpm run smoke:init and pnpm run smoke:launch passed.
  • pnpm dev registered https://hyperwiki.localhost:1355 with the current fallback proxy and the workspace returned HTTP 200 through Portless.
