Roadmap

  1. Claim Localhost Tooling as the product category and trust-boundary model.
  2. Establish the CLI, config, HTML wiki scaffold, and local-only server.
  3. Serve a useful static hyperwiki workspace over the repo wiki.
  4. Add real wterm-backed PTY panels and dogfood this repository.
  5. Add a one-command launch flow that starts the local server, opens the browser workspace, and prepares the default wterm operating session.
  6. Make verification loops a first-class product primitive across checks, tests, reviews, screenshots, CI status, and manual evidence.
  7. Add an incubation layer for free-form ideas that can become registered local hyperwiki projects.
  8. Follow with structured MCP integration around project contracts, current plans, source context, ideas, verification loops, guardrails, and controlled prompt submission.
