Roadmap

  1. Claim Localhost Tooling as the product category and trust-boundary model.
  2. Establish the CLI, config, HTML wiki scaffold, and local-only server.
  3. Serve a useful static hyperwiki workspace over the repo wiki.
  4. Add real wterm-backed PTY panels and dogfood this repository.
  5. Ship the Tauri desktop app as the canonical no-browser-chrome workspace with Rust-owned local runtime behavior and compatibility CLI commands.
  6. Make verification loops a first-class product primitive across checks, tests, reviews, screenshots, CI status, and manual evidence.
  7. Keep project creation direct: imported briefs initialize registered local hyperwiki projects without an idea incubation layer.
  8. Dogfood the read-only MCP transport and define permissioned action tools for review workflow execution and controlled prompt submission.
