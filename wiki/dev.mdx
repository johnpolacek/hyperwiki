<h1>Development</h1>
<h2>Commands</h2>
<pre><code>pnpm install
pnpm run check
pnpm run check:all
pnpm run tauri:build
pnpm wt:doctor
pnpm run smoke:tauri-static-assets
pnpm dev
src-tauri/target/release/hyperwiki init --yes
src-tauri/target/release/hyperwiki reset --dry-run
src-tauri/target/release/hyperwiki</code></pre>
<h2>Desktop Runtime</h2>
<ul>
  <li>The canonical app runtime is the Tauri bundle at <code>src-tauri/target/release/bundle/macos/hyperwiki.app</code>.</li>
  <li>The Rust binary provides commands for <code>init</code>, <code>reset</code>, <code>dev</code>, <code>launch</code>, <code>mcp</code>, and <code>wt</code>.</li>
  <li><code>pnpm dev</code> runs <code>tauri dev</code>; hyperwiki no longer has a browser-server development runtime.</li>
  <li>The Vite React app is the only app shell. Static files under <code>public/</code> are limited to package-served assets such as <code>/assets/wiki.css</code>, <code>/assets/theme.css</code>, favicon, and docs fonts.</li>
</ul>
<h2>Project Previews</h2>
<ul>
  <li>Portless remains the preferred preview mechanism for user projects opened inside hyperwiki.</li>
  <li>For every hyperwiki project, <code>pnpm dev</code> is the canonical local command. If a project has frontend, backend, workers, or other long-running processes, the package <code>dev</code> script should orchestrate them, commonly with <code>concurrently</code>, while Hyperwiki still runs only <code>pnpm dev</code>.</li>
  <li>Feature worktree preview URLs should continue to use <code>https://&lt;branch-slug&gt;.hyperwiki.localhost</code> when a project provides a Portless-backed preview.</li>
  <li>hyperwiki's own development runtime is the Tauri desktop app, not a Portless browser route.</li>
</ul>
<h2>Parallel Worktrees</h2>
<pre><code>pnpm wt:doctor
pnpm wt:create feature/my-change
pnpm wt:list
pnpm wt:resume feature/my-change
pnpm wt:open feature/my-change
pnpm wt:finish feature/my-change
pnpm wt:prune</code></pre>
<p>Feature worktrees live under <code>../hyperwiki.worktrees/&lt;branch-slug&gt;</code>. The finish policy is merge, preserving feature branch history.</p>
<h2>Packaging Notes</h2>
<p><code>pnpm run tauri:build</code> currently verifies the macOS <code>.app</code> target. DMG generation is intentionally deferred until signing/notarization and installer diagnostics are handled as release work.</p>
<h2>Workflow</h2>
<p>Plan meaningful work in <a href="/wiki/plans/index.mdx">plans</a>, keep source truth in repo files and Git, and use the dev workspace to inspect wiki pages and terminal sessions.</p>
